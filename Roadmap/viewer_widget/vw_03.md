# VW-03 Completion Notes

## Status

Status: completed

`VW-03` is now implemented.

The Harpy `Viewer` widget is available in napari, is wired to shared
`HarpyAppState`, supports coordinate-system-first browsing, and can load:

- labels layers
- stack images
- overlay images with selected channels and colors
- `SpatialData` zarr stores through the widget itself

In addition, `napari-harpy` now contributes a napari reader for SpatialData
zarr stores, so command-line opening can flow directly into Harpy state and the
Harpy dock widgets.

## Final Outcome

The implemented `Viewer` workflow now provides:

- shared `sdata` state through `HarpyAppState`
- refresh on `sdata_changed`
- one `Viewer` dock widget registered in `napari.yaml`
- automatic inclusion of `Viewer` in `Interactive(...)`
- coordinate-system-first browsing of images and segmentations
- labels `Add / Update in viewer`
- image `Add / Update in viewer` in:
  - `stack`
  - `overlay`
- per-channel overlay color selection
- `Load SpatialData` from the widget UI
- a napari reader that can:
  - claim a SpatialData `.zarr`
  - load it into Harpy shared state
  - open the Harpy dock widgets
  - keep `Viewer` as the active visible tab

## Post-Slice Refinements

During implementation, a few additional refinements were completed that are now
part of the finished `VW-03` result:

- long element names are elided in image and labels cards, with tooltip on
  truncation
- long success messages shorten visible element names while preserving the full
  message in the tooltip when needed
- overlay channel lists are scrollable and capped to a small visible height
- empty overlay selection removes the currently loaded visualization for that
  image and still surfaces the validation message
- the reader opens all Harpy dock widgets when a SpatialData store is opened
  through napari, in the order:
  - `Viewer`
  - `Feature Extraction`
  - `Object Classification`
- `Viewer` is kept as the active tab in that reader-driven flow

## Acceptance Summary

`VW-03` now satisfies the intended first-version viewer workflow:

- [x] the new `Viewer` dock widget is available in napari
- [x] the widget can load a `SpatialData` zarr store
- [x] changing the loaded `sdata` updates the widget through `sdata_changed`
- [x] coordinate systems are listed from the loaded `sdata`
- [x] labels and images are filtered by the selected coordinate system
- [x] a selected labels element can be loaded into napari
- [x] a selected image can be loaded in stack mode
- [x] selected channels can be loaded in overlay mode
- [x] napari command-line opening can feed a SpatialData store into Harpy state
